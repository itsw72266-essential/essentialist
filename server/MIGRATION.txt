This directory is the **Express** reference implementation (same domain logic as
`client/src/fullstack/` + Route Handlers under `/api/next/*`).

Next.js storefront runtime:
  - **Production API** is served by the **Next app** (`client/`) at **`/api/next/*`**.
  - Browser HTTP uses **`@/lib/apiClient`** (axios + cookies + refresh); errors use **`@/lib/axiosToastError`**.
  - See **docs/next-fullstack-migration.txt** for the full map of routes and env (`MONGODB_URI`, JWT secrets, Payunit, Stripe webhooks, etc.).

Express (`server/`) is **optional** for the live site: keep it for scripts, local tools,
or a separate process. It is **not** required when `NEXT_PUBLIC_API_URL` / `API_URL`
point at the Next deployment.

Cutover reminders (unchanged behavior vs Express):
  - Stripe webhook: **`/api/next/order/webhook`**
  - Payunit webhook: **`/api/next/payments/webhook`** (or `PAYUNIT_NOTIFY_URL`)
  - Guest COD / guest orders: **`POST /api/next/order/guest-cod`**; alias **`POST /api/guest-orders`**

Do not remove routes here without verifying Next parity. New behavior should land in
`client/src/fullstack/` first, then Express can follow or stay frozen for reference.
