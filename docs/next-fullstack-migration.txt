Essentialist — Next.js full-stack migration (incremental)

Are we moving `server/` (Express) into Next.js?
  You asked for a **complete full-stack Next.js app** without relying on an
  external Express process for those routes, while **`server/` stays intact**
  (reference + reuse for another app). The approach is **dual track**:
  - **`server/`** — unchanged; keep committing and running it as today.
  - **`client/src/fullstack/`** — new home for Zod schemas, Mongoose models,
    controllers, and Route Handlers that implement the same behavior against
    the **same `MONGODB_URI`** (and shared secrets like `RESEND_API`) when you
    wire traffic to `/api/next/...`.
  - **`client/src/backend/`** — legacy HTTP surface + optional **proxies** to
    Express (`/api/legacy/...`) during transition.

  Reviews path notes (Next cannot mount two sibling `[id]` dynamic segments; a few URLs differ):
    - Product-scoped list + delete-my-review: GET|DELETE /api/next/reviews/product/[productId]
      (Express: GET|DELETE /api/reviews/:productId)
    - Comments: GET|POST /api/next/reviews/review/[reviewId]/comments
      (Express: /api/reviews/:reviewId/comments)
    - Other paths match Express under /reviews/... (public, public/product/..., id/..., admin/..., comments/...).

  Cutover: `SummaryApi` and Axios now target **`/api/next/...`** on the same
  origin. Set `NEXT_PUBLIC_API_URL` and `API_URL` to your Next origin (e.g.
  `http://localhost:3000`) so server components and `next build` self-fetch hit
  Route Handlers, not Express. Stripe webhooks must use **`/api/next/order/webhook`**.

  Env: same MongoDB + JWT secrets as Express; `requireAdmin` routes need role ADMIN.
  Review admin routes use `requireAdminPrivilege` (ADMIN or SUPER_ADMIN). Auth reads
  `accessToken` cookie first, then `Authorization: Bearer` (matches Express `auth`).
  Payunit notify URL: set `PAYUNIT_NOTIFY_URL` explicitly, or rely on `BACKEND_URL` /
  `FRONTEND_URL` (see `/api/next/payments/*` note above).

  Note: Express Redis cache middleware is a no-op in Next (`cacheNoop.js`). Express rate
  limiters, idempotency keys, duplicate-order guards, and multer are not wired on Next;
  parity is behavioral for controllers only.

  Stripe: `fullstack/config/stripe.js` uses `new Stripe(secret)` (Stripe SDK v22). Webhook
  route verifies signature with raw body then calls the same `webhookStripe` handler.

  Payunit: `fullstack/config/payunit.js` exposes `getPayunitClient()` (lazy; returns `null`
  if `PAYUNIT_API_KEY`, `PAYUNIT_API_USERNAME`, and `PAYUNIT_API_PASSWORD` are unset so
  `next build` does not throw). Optional `PAYUNIT_MODE` (default `test`). Install
  `@payunit/nodejs-sdk` in the client app.
